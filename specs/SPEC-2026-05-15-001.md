# Agent-Ready Specification: Publish agentic-sdlc to PyPI

## 1. Context (Business Problem)

### Current State

`agentic-sdlc` CLI is functional (181 tests, 86.79% coverage) but install requires cloning the repo and running `pip install -e ".[dev]"`. No published package exists on PyPI. README instructs `pip install agentic-sdlc` but this fails — the package does not exist on the registry.

### Desired State

`pip install agentic-sdlc` works from any machine without cloning the repo. Package is live on PyPI at `https://pypi.org/project/agentic-sdlc/`. GitHub Actions publishes new versions automatically on tag push.

### Business Value

- Framework adoption requires zero repo setup for end users
- Standard Python install UX (`pip install`)
- Enables PyPI download badge in README

---

## 2. Goal State (Definition of Done)

- [ ] `pyproject.toml` has all required PyPI metadata (description, classifiers, homepage, keywords)
- [ ] Package builds cleanly: `python -m build` produces `.tar.gz` and `.whl` in `dist/`
- [ ] `twine check dist/*` passes with no errors
- [ ] `.github/workflows/publish.yml` publishes to PyPI on `v*` tag push using Trusted Publishing (OIDC, no API token stored)
- [ ] README Quick Start `pip install agentic-sdlc` reflects the published package
- [ ] Version in `src/agentic_sdlc/__init__.py` and `pyproject.toml` match

## 3. Non-Negotiable Constraints

- Use PyPI Trusted Publishing (OIDC) — no `PYPI_API_TOKEN` secret in repo
- Do not publish from local machine — CI/CD only
- `python -m build` must succeed with no warnings
- `data_classification: public` — no sensitive data in package metadata


---

## 5. Edge Cases & Error Handling

- Version conflict: if `v0.1.0` already exists on PyPI, workflow must fail loudly (not silently skip)
- Build failure: `python -m build` exit code != 0 must block publish step
- Trusted Publishing misconfiguration: OIDC token fetch failure must surface the error, not fall back to password auth

---

## 6. Rollback Conditions

PyPI does not support unpublishing a released version. Rollback means:

- Tag deletion on GitHub does not retract a published release from PyPI.
- If a bad build is published, release a patch version (e.g., `v0.1.1`) with the fix.
- Workflow failure before `pypa/gh-action-pypi-publish` is safe — nothing reaches PyPI.
- If OIDC token fetch fails, the workflow exits 1 and no publish occurs.

**Prevention:** `twine check dist/*` step blocks publish on malformed packages.

### Workflow file: `.github/workflows/publish.yml`

```yaml
name: Publish to PyPI

on:
  push:
    tags: ["v*"]

jobs:
  publish:
    runs-on: ubuntu-latest
    environment: pypi
    permissions:
      id-token: write
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install build twine
      - run: python -m build
      - run: twine check dist/*
      - uses: pypa/gh-action-pypi-publish@release/v1
```

### PyPI Trusted Publishing setup (one-time, human task)

1. Go to https://pypi.org/manage/account/publishing/
2. Add pending publisher: `bigknoxy/agentic-sdlc-framework`, workflow `publish.yml`, environment `pypi`
3. Create GitHub environment `pypi` in repo settings
